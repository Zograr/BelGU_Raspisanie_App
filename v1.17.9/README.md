# v1.17.9

Хотфикс установки обновлений:
- исправлена главная причина ошибки «Не удалось установить приложение» при обновлении из самого приложения;
- `ApkFileProvider` использовал неправильное имя SharedPreferences (`schedule_prefs` вместо `schedule_cache`);
- из-за этого установщик получал `content://...apkprovider/update.apk`, но провайдер не находил реальный APK-файл;
- теперь `ApkFileProvider` читает тот же `schedule_cache`, что и MainActivity;
- добавлен fallback-поиск последнего APK в папках `updates`, если путь не сохранился.

Сохранено:
- улучшенный интерфейс оценок из v1.17.8;
- проверка валидности APK;
- кастомные карточки оценок;
- фиксы WebView/журнала СПО.

Важно:
- если ты сейчас на v1.17.7 или v1.17.8, этот фикс всё равно лучше поставить вручную один раз, потому что ошибка находится в уже установленном старом `ApkFileProvider`.
- после установки v1.17.9 следующие обновления должны открываться через установщик стабильнее.

Известные баги:
- функция оценок всё ещё зависит от HTML таблицы dekanat.bsuedu.ru. Если таблица отличается, парсер может потребовать ещё один хотфикс.
